# 캘린더 앱 예약 흐름 디자인

팀 캘린더 기반의 회의 예약 흐름을 프로토타이핑한 웹 애플리케이션입니다.

Vercel URL: https://calender-coral-alpha.vercel.app

## 프로젝트 소개

이 프로젝트는 팀원별 일정, 외근, 회의 기피 시간, 참석자 상태를 한 화면에서 확인하며 회의를 생성할 수 있는 캘린더 UI 프로토타입입니다.

Google Calendar와 유사한 주간 캘린더 경험을 기반으로, 팀 단위 회의 예약에 필요한 정보들을 더 명확하게 보여주는 것을 목표로 했습니다.

주요 기능은 다음과 같습니다.

- 주간 캘린더에서 팀원별 일정 확인
- 회의 생성 모달을 통한 일정, 프로젝트, 회의 유형, 참석자 설정
- 참석자별 프로필, 역할, 참석 가능 시간 표시
- 외근 및 회의 기피 시간 표시
- 회의 상세 모달에서 참석자, 회의실/위치, 설명 확인
- 팀원 캘린더 표시/숨김, 색상 변경, 팀원 추가/삭제

## 문제 정의

팀 단위로 회의를 잡을 때는 단순히 빈 시간만 보는 것으로는 부족합니다.

실제 회의 예약 과정에서는 다음과 같은 정보가 함께 필요합니다.

- 각 팀원의 기존 일정
- 참석자의 역할과 참여 필요 여부
- 외근 여부
- 점심 직후, 오전 이른 시간, 늦은 오후처럼 회의를 피하고 싶은 시간
- 새 회의가 생성되었을 때 참석자별 수락/미수락 상태
- 회의실 또는 위치 정보

기존 캘린더 UI에서는 이런 맥락이 분산되어 있어 회의 조율 과정이 번거로울 수 있습니다. 이 프로토타입은 회의 생성 흐름 안에서 필요한 정보를 한 번에 확인하고, 더 자연스럽게 일정을 조율할 수 있도록 설계했습니다.

## 실행 방법

의존성 설치:

```bash
pnpm install
```

개발 서버 실행:

```bash
pnpm run dev
```

브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:5173
```

프로덕션 빌드:

```bash
pnpm run build
```
